GUIDING CONSTRAINTS (non-negotiable)
--------------------------------------

- Cost must be as close to zero as possible. Client said "little to nothing."

SECTION 6: THE UPDATE WORKFLOW (Core Pain Point)
--------------------------------------------------
This is the client's most-repeated request across the entire video and email.

  "I want to make updates, changes, add things with just telling somebody
   or emailing them or texting them. I don't want to go into Manus and have
   to do all the updates myself."

The solution for Version 1 is simple and free:
  - All site content lives in one file: client/src/lib/data.ts
  - The intermediary (whoever manages the site) receives a text/email from the client
    such as "add this book: Atomic Habits by James Clear" and updates that one file
  - A new build is deployed to Vercel/Netlify (one command or automatic via GitHub)
  - No admin panel, no backend, no login required

This workflow costs nothing and requires no new infrastructure.
An AI-assisted pipeline (auto-parsing emails into data changes) can be explored later
if the update volume justifies it — but it is NOT what the client is asking for now.
